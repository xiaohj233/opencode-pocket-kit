# 代理配置

代理配置文件位于根目录：

```text
proxy.conf
```

默认不开启代理，端口为空。

## 默认配置

```ini
PROXY_ENABLED=0
PROXY_HOST=127.0.0.1
PROXY_PORT=
PROXY_SCHEME=http
PROXY_URL=
```

如果你的网络可以直接访问 GitHub 和 npm registry，请保持默认值。

## 启用本地代理

以本地 HTTP 代理端口 `7897` 为例：

```ini
PROXY_ENABLED=1
PROXY_HOST=127.0.0.1
PROXY_PORT=7897
PROXY_SCHEME=http
PROXY_URL=
```

如果已经有完整代理地址，也可以直接填写：

```ini
PROXY_ENABLED=1
PROXY_URL=http://127.0.0.1:7897
```

`PROXY_URL` 优先级高于 `PROXY_HOST`、`PROXY_PORT`、`PROXY_SCHEME`。

## 字段说明

| 字段 | 默认值 | 说明 |
|---|---|---|
| `PROXY_ENABLED` | `0` | 是否启用代理。`1` 表示启用，`0` 表示关闭 |
| `PROXY_HOST` | `127.0.0.1` | 本地代理主机 |
| `PROXY_PORT` | 空 | HTTP 代理端口 |
| `PROXY_SCHEME` | `http` | 代理协议，通常是 `http` |
| `PROXY_URL` | 空 | 完整代理地址，填写后优先生效 |
| `NPM_REGISTRY` | `https://registry.npmjs.org` | npm 源地址 |
| `GIT_CLONE_TIMEOUT_SECONDS` | `600` | Git clone 超时时间 |
| `GIT_LSREMOTE_TIMEOUT_SECONDS` | `90` | Git 远程探测超时时间 |
| `GIT_LOW_SPEED_LIMIT` | `1000` | Git 低速阈值 |
| `GIT_LOW_SPEED_TIME_SECONDS` | `180` | Git 低速判定时间 |
| `NPM_PACKAGE_TIMEOUT_SECONDS` | `900` | npm 包安装超时时间 |
| `INSTALL_OMO` | `1` | 是否安装 OMO |
| `CLEAN_OLD_LOGS_ON_INSTALL` | `1` | 安装前是否清理旧日志 |
| `OMO_RUN_DOCTOR` | `0` | 安装流程中是否运行 OMO Doctor |
| `GIT_VERBOSE` | `0` | 是否输出 Git 详细网络日志 |

## 常见 HTTP 代理端口

| 软件 | 常见 HTTP 端口 |
|---|---|
| Clash / Mihomo | `7890`、`7897` |
| v2rayN | `10809` |
| 自定义代理 | 以软件实际显示为准 |

注意：很多代理软件同时提供 SOCKS 和 HTTP 端口。Git 和 npm 通常需要 HTTP 代理地址。不要把 SOCKS 端口直接填入 `PROXY_PORT`，除非代理软件明确说明该端口是 HTTP 或 mixed 混合端口。

## npm 源

默认使用官方源：

```ini
NPM_REGISTRY=https://registry.npmjs.org
```

如果 npm 官方源访问较慢，可以临时改为镜像源，例如：

```ini
NPM_REGISTRY=https://registry.npmmirror.com
```

切换镜像源后，如果遇到包版本滞后或安装异常，可以改回官方源再试。

## 代理作用范围

安装和下载脚本会把代理用于：

- GitHub 仓库访问。
- Git sparse checkout。
- npm 包安装。
- PowerShell 网络请求。

启动 OpenCode 时，脚本也会继承当前便携环境变量；但 OpenCode 内部具体网络请求是否使用代理，还取决于 OpenCode、插件或模型供应商 SDK 本身的代理支持。

## 排错步骤

### 1. 确认代理软件正在运行

先打开代理软件，确认 HTTP 代理端口可用。

### 2. 确认端口填写正确

例如代理软件显示 HTTP 端口为 `7897`，则写：

```ini
PROXY_ENABLED=1
PROXY_HOST=127.0.0.1
PROXY_PORT=7897
PROXY_SCHEME=http
```

### 3. 运行安装或更新

```bat
安装.cmd
```

或：

```bat
更新.cmd
```

脚本会在日志中显示是否启用代理。

### 4. 查看日志

日志位于：

```text
logs\
```

网络下载相关日志通常是：

```text
network-*.log
```

### 5. 调高超时

网络较慢时，可以调高：

```ini
GIT_CLONE_TIMEOUT_SECONDS=900
GIT_LSREMOTE_TIMEOUT_SECONDS=180
NPM_PACKAGE_TIMEOUT_SECONDS=1200
```

### 6. 开启 Git 详细日志

需要排查 Git 连接问题时，可以临时设置：

```ini
GIT_VERBOSE=1
```

完成排查后建议改回：

```ini
GIT_VERBOSE=0
```
