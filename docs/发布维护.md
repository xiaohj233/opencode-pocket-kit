# 发布维护

本文说明如何维护和发布 OpenCode Pocket Kit。

## 仓库建议

推荐仓库名：

```text
opencode-pocket-kit
```

推荐发布包名：

```text
opencode-pocket-kit-V1.0.7.zip
```

## 版本文件

版本号写在根目录：

```text
VERSION
```

当前发布版：

```text
V1.0.7
```

## 编码规范

| 文件类型 | 编码 | 换行 | 说明 |
|---|---|---|---|
| `.cmd` | ASCII | CRLF | 入口批处理脚本 |
| `.ps1` | UTF-8 with BOM | CRLF | PowerShell 脚本 |
| `.md` | UTF-8 | LF 或 CRLF | GitHub 文档 |
| `.json` / `.jsonc` | UTF-8 | LF 或 CRLF | 配置文件 |
| `.conf` | UTF-8 或 ASCII | LF 或 CRLF | 简单键值配置 |

## 发布前检查

发布前建议确认：

1. 根目录没有 `vault\secrets.env.enc`。
2. 根目录没有真实运行日志。
3. `proxy.conf` 默认关闭代理，端口为空。
4. `opencode.json` 不包含个人模型供应商密钥。
5. `README.md` 和 `docs\` 不包含本地调试记录。
6. 根目录 `.cmd` 文件没有被保存成带 BOM 或乱码格式。
7. `scripts\*.ps1` 是 UTF-8 with BOM。
8. `诊断.cmd` 能正常运行。
9. `安装.cmd` 能在新目录中完成安装流程。
10. `.gitignore` 能排除密钥、日志、缓存和安装后的大体积依赖。

## 不应提交的内容

`.gitignore` 应排除：

```text
vault/secrets.env.enc
logs/*.log
backups/*
repo-cache/*
npm-global/*
config/opencode/node_modules/*
bin/opencode.exe
.portable-state/*
```

这些内容属于本地运行时文件，不适合提交到公开仓库。

## 应保留的占位文件

为了让 Git 保留空目录，可以保留 `.keep` 文件，例如：

```text
logs/.keep
vault/.keep
repo-cache/.keep
projects/.keep
```

## 推荐发布流程

1. 新建干净目录。
2. 放入源码和脚本。
3. 检查 `proxy.conf` 默认值。
4. 检查 `command.conf` 默认命令名。
5. 检查 `README.md` 和 `docs\`。
6. 初始化 Git 仓库。
7. 推送到 GitHub。
8. 创建 `V1.0.7` 标签。
9. 创建 GitHub Release。
10. 上传 `opencode-pocket-kit-V1.0.7.zip`。

可以使用单独的 GitHub 发布脚本完成初始化仓库、推送、打标签和创建 Release。

## 文档维护原则

文档应遵循：

- 不包含个人 API Key。
- 不包含本地绝对路径，除非是示例路径。
- 不包含历史调试日志。
- 不把上游项目行为写成无法验证的结论。
- 涉及 OpenCode、OMO、Skills 的行为时，优先参考官方文档或实际脚本。
- 新增脚本或配置文件后，同步更新 `目录结构.md` 和 `配置说明.md`。
- 新增插件或 Skills 后，同步更新 `组件清单.md`。

## 测试建议

发布前至少测试：

```bat
安装.cmd
诊断.cmd
编辑密钥.cmd
启动.cmd
注册命令.cmd
opkcode
注销命令.cmd
```

`编辑密钥.cmd` 测试时可以使用假的示例值，不要使用真实 API Key 打包发布。
