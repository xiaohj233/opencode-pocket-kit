# 快速开始

本文说明如何从零开始使用 OpenCode Pocket Kit。

## 环境要求

使用前请确认电脑上已经具备以下工具：

| 工具 | 说明 |
|---|---|
| Windows 10 或 Windows 11 | 本包面向 Windows 设计 |
| PowerShell 5.1 或更高版本 | Windows 10/11 默认通常已包含 |
| Node.js 和 npm | 用于安装 `opencode-ai`、OMO 和相关插件 |
| Git | 用于下载可选 Skills 和 Superpowers |
| 网络访问能力 | 需要访问 npm registry 和 GitHub，不能直连时请配置代理 |

可选工具：

| 工具 | 是否必需 | 说明 |
|---|---|---|
| GitHub CLI `gh` | 否 | 只影响 GitHub 自动化能力，不影响 OpenCode 基础启动 |
| bun / bunx | 否 | 本包不依赖 bun 或 bunx |
| 本地代理软件 | 否 | 网络无法直连 GitHub/npm 时使用 |

## 第一次使用

### 1. 解压目录

将压缩包解压到任意目录，例如：

```text
D:\Tools\opencode-pocket-kit
```

建议整体移动整个文件夹，不要只移动内部子目录。便携包内很多路径都是相对包根目录计算的，只要整体移动，通常不需要重新配置。

路径中可以包含中文和空格。本包启动脚本会尽量使用 `-LiteralPath` 和带引号路径处理。不过如果遇到第三方工具自身无法处理复杂路径，可以尝试移动到较短路径，例如：

```text
D:\Tools\opencode-pocket-kit
```

### 2. 配置代理

默认不开启代理。如果你的网络可以直接访问 GitHub 和 npm registry，不需要修改。

如果需要代理，打开根目录：

```text
proxy.conf
```

填写 HTTP 代理端口。例如本地 HTTP 代理端口是 `7897`：

```ini
PROXY_ENABLED=1
PROXY_HOST=127.0.0.1
PROXY_PORT=7897
PROXY_SCHEME=http
```

注意：这里需要 HTTP 代理端口。SOCKS 端口不能直接当作 HTTP 端口使用，除非代理软件明确提供 HTTP 混合端口。

详细说明见 [代理配置](./代理配置.md)。

### 3. 安装组件

双击运行：

```bat
安装.cmd
```

安装流程会执行以下工作：

1. 检查系统依赖。
2. 读取 `proxy.conf`。
3. 安装或更新 OpenCode 本体 `opencode-ai`。
4. 准备 `bin\opencode.exe`。
5. 安装 Oh-My-OpenAgent。
6. 安装或同步 `config\opencode` 目录中的 Node 依赖。
7. 运行 OMO 官方安装器。
8. 写入或修复 `opencode.json` 中的 OMO 插件项。
9. 写入或修复 `tui.json` 中的 TUI 插件项。
10. 安装 Comment Checker。
11. 测试 GitHub/npm 网络。
12. 下载并安装可选 Skills 和 Superpowers。

安装日志位于：

```text
logs\
```

常见日志文件名类似：

```text
install-20260530-120029.log
network-20260530-120041.log
```

### 4. 编辑密钥

双击运行：

```bat
编辑密钥.cmd
```

按 `KEY=VALUE` 格式输入密钥，每行一个变量，例如：

```env
DEEPSEEK_API_KEY=sk-xxxx
OPENROUTER_API_KEY=sk-or-v1-xxxx
TAVILY_API_KEY=tvly-xxxx
```

输入空行结束，然后设置密钥库密码。脚本会生成：

```text
vault\secrets.env.enc
```

该文件是加密密钥库，不是明文 `.env`。

### 5. 启动 OpenCode

双击运行：

```bat
启动.cmd
```

默认工作目录是：

```text
projects\
```

如果存在 `vault\secrets.env.enc`，启动脚本会要求输入密钥库密码。解密成功后，脚本会把密钥临时注入当前进程环境变量，然后启动：

```text
bin\opencode.exe
```

如果没有密钥库，启动脚本仍会启动 OpenCode，只是不会注入 API Key。

### 6. 诊断问题

遇到问题时双击运行：

```bat
诊断.cmd
```

诊断会检查：

- 便携环境变量。
- OpenCode 配置文件。
- OMO 路由配置候选文件。
- node、npm、git、opencode、gh、bun、bunx 等工具。
- OpenCode 版本。
- OMO 插件配置。
- TUI 插件配置。
- Comment Checker。
- OMO Doctor 输出。

## 命令行启动方式

如果你希望在任意终端中像输入 `opencode` 一样启动本包，可以先运行：

```bat
注册命令.cmd
```

默认注册命令名是：

```bat
opkcode
```

重新打开终端后，进入项目目录并运行：

```bat
cd D:\Code\my-project
opkcode
```

这时 OpenCode 会以当前终端目录作为工作目录启动。

详细说明见 [命令行启动](./命令行启动.md)。

## 日常使用流程

### 普通启动

```bat
启动.cmd
```

### 修改密钥

```bat
编辑密钥.cmd
启动.cmd
```

### 在项目目录启动

```bat
cd D:\Code\my-project
opkcode
```

### 更新组件

```bat
更新.cmd
```

### 诊断环境

```bat
诊断.cmd
```

## 安装成功后应看到什么

安装成功后，目录中通常会出现：

```text
bin\opencode.exe
npm-global\node_modules\
config\opencode\opencode.json
config\opencode\tui.json
config\opencode\package.json
config\opencode\node_modules\
config\opencode\skills\
logs\
```

如果 `诊断.cmd` 中只提示 GitHub CLI 未登录，这通常不影响 OpenCode 本体和 OMO 使用。只有需要 GitHub 自动化时才需要运行：

```bat
gh auth login
```
