# 代理配置

代理配置文件位于根目录：

```text
proxy.conf
```

默认不开启代理，端口为空。

## 默认配置

```ini
PROXY_ENABLED=0
PROXY_HOST=127.0.0.1
PROXY_PORT=
PROXY_SCHEME=http
PROXY_URL=
```

如果你的网络可以直接访问 GitHub 和 npm registry，请保持默认值。

## 启用本地代理

以本地 HTTP 代理端口 `7897` 为例：

```ini
PROXY_ENABLED=1
PROXY_HOST=127.0.0.1
PROXY_PORT=7897
PROXY_SCHEME=http
PROXY_URL=
```

如果你已经有完整代理地址，也可以直接填写：

```ini
PROXY_ENABLED=1
PROXY_URL=http://127.0.0.1:7897
```

`PROXY_URL` 优先级高于 `PROXY_HOST`、`PROXY_PORT`、`PROXY_SCHEME`。

## 常见端口

| 软件 | 常见 HTTP 端口 |
|---|---|
| Clash / Mihomo | `7890`、`7897` |
| v2rayN | `10809` |
| 自定义代理 | 以软件实际显示为准 |

注意：Git 和 npm 需要 HTTP 代理地址。SOCKS 端口不能直接当作 HTTP 端口使用，除非你的代理软件明确提供 HTTP 混合端口。

## npm 源

默认使用官方源：

```ini
NPM_REGISTRY=https://registry.npmjs.org
```

如果需要国内镜像，可以改为：

```ini
NPM_REGISTRY=https://registry.npmmirror.com
```

## 超时配置

```ini
GIT_CLONE_TIMEOUT_SECONDS=600
GIT_LSREMOTE_TIMEOUT_SECONDS=90
GIT_LOW_SPEED_LIMIT=1000
GIT_LOW_SPEED_TIME_SECONDS=180
NPM_PACKAGE_TIMEOUT_SECONDS=900
```

网络较慢时可以适当提高这些值。

## 安装行为

```ini
INSTALL_OMO=1
CLEAN_OLD_LOGS_ON_INSTALL=1
OMO_RUN_DOCTOR=0
```

| 字段 | 说明 |
|---|---|
| `INSTALL_OMO` | 是否安装 OMO |
| `CLEAN_OLD_LOGS_ON_INSTALL` | 安装前是否清理旧日志 |
| `OMO_RUN_DOCTOR` | 安装流程中是否运行 OMO Doctor |

## 排错步骤

1. 运行 `诊断.cmd`。
2. 确认 `proxy.conf` 中的端口是 HTTP 代理端口。
3. 确认代理软件正在运行。
4. 如果 GitHub 可访问但下载慢，提高 `GIT_CLONE_TIMEOUT_SECONDS`。
5. 如果 npm 下载失败，尝试临时切换 `NPM_REGISTRY`。
