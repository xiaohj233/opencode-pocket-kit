# 配置说明

本项目主要涉及 OpenCode、OMO、TUI、代理和密钥几类配置。

## OpenCode 主配置

文件位置：

```text
config\opencode\opencode.json
```

默认内容：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": [
    "oh-my-openagent"
  ]
}
```

这里不默认写入 provider、model、small_model。模型路由优先交给 OMO 和用户自己的配置处理。

## TUI 配置

文件位置：

```text
config\opencode	ui.json
```

默认内容：

```json
{
  "plugin": [
    "oh-my-openagent/tui"
  ]
}
```

该配置用于启用 OMO 的 TUI 侧边栏和 TUI 命令。

## OMO 配置

OMO 官方安装器会生成：

```text
config\opencode\oh-my-openagent.json
```

也可能识别这些候选文件：

```text
config\opencode\oh-my-openagent.jsonc
config\opencode\oh-my-opencode.json
config\opencode\oh-my-opencode.jsonc
projects\.opencode\oh-my-openagent.json
projects\.opencode\oh-my-openagent.jsonc
```

全局配置放在 `config\opencode`。项目级配置放在项目目录的 `.opencode` 文件夹。

## Node 依赖配置

文件位置：

```text
config\opencode\package.json
```

默认声明：

```json
{
  "dependencies": {
    "oh-my-openagent": "latest",
    "@opencode-ai/plugin": "latest",
    "@code-yeongyu/comment-checker": "latest"
  }
}
```

安装脚本会在 `config\opencode` 中执行 npm install，确保 OpenCode 插件解析时能找到依赖。

## 代理配置

文件位置：

```text
proxy.conf
```

默认关闭代理。详情见 [代理配置](./代理配置.md)。

## 密钥配置

文件位置：

```text
vault\secrets.env.enc
```

使用 `编辑密钥.cmd` 创建。详情见 [密钥安全](./密钥安全.md)。

## 环境变量重定向

启动时会临时设置：

| 变量 | 指向 |
|---|---|
| `OPENCODE_CONFIG` | `config\opencode\opencode.json` |
| `OPENCODE_CONFIG_DIR` | `config\opencode` |
| `APPDATA` | `config` |
| `USERPROFILE` | `home` |
| `HOME` | `home` |
| `NODE_PATH` | 便携 node_modules 路径 |
| `PATH` | 便携 bin、npm-global、node_modules bin 路径 |

这些环境变量只对当前启动进程有效。


## 命令行别名配置

命令行别名由根目录 `command.conf` 控制：

```ini
COMMAND_NAME=opkcode
```

运行 `注册命令.cmd` 后，会在 `cmd/` 目录生成对应的 `.cmd` 文件，并把 `cmd/` 目录加入当前用户的 `Path`。

在任意终端中运行该命令时，OpenCode 会以当前终端所在目录作为工作目录启动。

详细说明见 [命令行启动](./命令行启动.md)。
