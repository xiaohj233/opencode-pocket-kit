# 发布维护

本文档说明如何维护和发布 OpenCode Pocket Kit。

## 仓库建议

推荐仓库名：

```text
opencode-pocket-kit
```

推荐发布包名：

```text
opencode-pocket-kit-v1.0.0.zip
```

## 版本文件

版本号写在根目录：

```text
VERSION
```

当前版本：

```text
V1.0.0
```

## 编码规范

| 文件类型 | 编码 | 换行 | 说明 |
|---|---|---|---|
| `.cmd` | ASCII | CRLF | 入口批处理脚本 |
| `.ps1` | UTF-8 with BOM | CRLF | PowerShell 脚本 |
| `.md` | UTF-8 | LF 或 CRLF | GitHub 文档 |
| `.json` | UTF-8 | LF 或 CRLF | 配置文件 |
| `.conf` | UTF-8 | LF 或 CRLF | 代理配置 |

## 发布前检查

发布前建议确认：

1. 根目录没有 `vault\secrets.env.enc`。
2. 根目录没有真实运行日志。
3. `proxy.conf` 默认关闭代理，端口为空。
4. `opencode.json` 不包含个人模型供应商密钥。
5. `README.md` 和 `docs/` 不包含本地调试记录。
6. `诊断.cmd` 能正常运行。
7. `安装.cmd` 能在新目录中完成安装流程。

## 不应提交的内容

`.gitignore` 应排除：

```text
vault/secrets.env.enc
logs/*.log
backups/*
repo-cache/*
npm-global/*
config/opencode/node_modules/*
bin/opencode.exe
.portable-state/*
```

## 推荐发布流程

1. 新建干净目录。
2. 放入源码和脚本。
3. 检查 `proxy.conf` 默认值。
4. 检查 README 和 docs。
5. 初始化 Git 仓库。
6. 推送到 GitHub。
7. 创建 `v1.0.0` 标签。
8. 创建 GitHub Release。
9. 上传 `opencode-pocket-kit-v1.0.0.zip`。

可以使用单独的 GitHub 发布脚本完成初始化仓库、推送、打标签和创建 Release。
