# 快速开始

## 环境要求

- Windows 10 或 Windows 11。
- PowerShell 5.1 或更高版本。
- Node.js 和 npm 可用。
- Git 可用。
- 网络可以访问 npm registry 和 GitHub。如果不能直连，请配置 `proxy.conf`。

## 第一次使用

### 1. 解压目录

将压缩包解压到任意目录，例如：

```text
D:\Tools\opencode-pocket-kit
```

目录可以移动。建议整体移动整个文件夹，不要只移动内部子目录。

### 2. 配置代理

默认不开启代理。如果你可以直连 GitHub 和 npm，不需要修改。

如果需要代理，打开根目录的 `proxy.conf`，填写 HTTP 代理端口：

```ini
PROXY_ENABLED=1
PROXY_HOST=127.0.0.1
PROXY_PORT=7897
PROXY_SCHEME=http
```

详细说明见 [代理配置](./代理配置.md)。

### 3. 安装组件

双击运行：

```bat
安装.cmd
```

安装流程会执行：

1. 安装 OpenCode 本体。
2. 安装 Oh-My-OpenAgent 和 TUI 插件。
3. 安装 Comment Checker。
4. 配置 OpenCode 和 TUI 插件入口。
5. 测试网络。
6. 下载并安装可选 Skills。

### 4. 编辑密钥

双击运行：

```bat
编辑密钥.cmd
```

按 `KEY=VALUE` 格式输入 API Key，例如：

```env
DEEPSEEK_API_KEY=sk-xxxx
OPENROUTER_API_KEY=sk-or-xxxx
```

输入空行结束，然后设置密钥库密码。

### 5. 启动 OpenCode

双击运行：

```bat
启动.cmd
```

如果存在 `vault\secrets.env.enc`，脚本会要求输入密码，并把密钥临时注入当前运行环境。

### 6. 诊断问题

遇到问题时运行：

```bat
诊断.cmd
```

诊断结果会检查：

- 便携环境变量。
- OpenCode 配置文件。
- OMO 路由配置候选。
- node、npm、git、opencode、gh 等工具。
- OpenCode 版本。
- Comment Checker。
- OMO Doctor。


## 命令行启动方式

如果你希望在任意终端中像输入 `opencode` 一样启动本包，可以先双击：

```bat
注册命令.cmd
```

默认注册的命令名是：

```bat
opkcode
```

重新打开终端后，进入项目目录并运行：

```bat
cd D:\Code\my-project
opkcode
```

OpenCode 会以当前终端目录作为工作目录启动。详细说明见 [命令行启动](./命令行启动.md)。

## 日常使用流程

常规流程：

```bat
启动.cmd
```

需要改密钥：

```bat
编辑密钥.cmd
启动.cmd
```

需要更新：

```bat
更新.cmd
```
