# 配置说明

本项目主要涉及 OpenCode、OMO、TUI、代理、密钥和命令行别名几类配置。

## OpenCode 主配置

文件位置：

```text
config\opencode\opencode.json
```

默认内容：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": [
    "oh-my-openagent"
  ]
}
```

这里默认只启用 OMO 插件，不默认写入 provider、model、small_model。模型供应商和 OMO 路由可以由用户根据自己的账号和模型服务自行配置。

## TUI 配置

文件位置：

```text
config\opencode\tui.json
```

默认内容：

```json
{
  "plugin": [
    "oh-my-openagent/tui"
  ]
}
```

该配置用于启用 OMO 的 TUI 侧边栏和 TUI 命令。

## OMO 配置

OMO 官方安装器会生成：

```text
config\opencode\oh-my-openagent.json
```

在重命名兼容期，OMO 也可能识别这些候选文件名：

```text
config\opencode\oh-my-openagent.jsonc
config\opencode\oh-my-opencode.json
config\opencode\oh-my-opencode.jsonc
```

项目级 OMO 配置可以放在：

```text
项目目录\.opencode\oh-my-openagent.json
项目目录\.opencode\oh-my-openagent.jsonc
```

建议优先让 OMO 官方安装器生成和维护全局路由配置。手动修改前，请先备份对应文件。

## Node 依赖配置

文件位置：

```text
config\opencode\package.json
```

默认声明：

```json
{
  "dependencies": {
    "oh-my-openagent": "latest",
    "@opencode-ai/plugin": "latest",
    "@code-yeongyu/comment-checker": "latest"
  }
}
```

安装脚本会在 `config\opencode` 中执行 npm install，确保 OpenCode 解析插件时能找到依赖。

## 代理配置

文件位置：

```text
proxy.conf
```

默认关闭代理。详情见 [代理配置](./代理配置.md)。

## 密钥配置

文件位置：

```text
vault\secrets.env.enc
```

使用 `编辑密钥.cmd` 创建。详情见 [密钥安全](./密钥安全.md)。

## 命令行别名配置

文件位置：

```text
command.conf
```

默认内容：

```ini
COMMAND_NAME=opkcode
```

运行：

```bat
注册命令.cmd
```

后，会在 `cmd\` 目录生成对应 `.cmd` 启动器，并把 `cmd\` 目录加入当前用户 `Path`。

## 环境变量重定向

启动时会临时设置以下变量：

| 变量 | 指向 |
|---|---|
| `OPENCODE_CONFIG` | `config\opencode\opencode.json` |
| `OPENCODE_CONFIG_DIR` | `config\opencode` |
| `APPDATA` | `config` |
| `USERPROFILE` | `home` |
| `HOME` | `home` |
| `NODE_PATH` | 便携 node_modules 路径 |
| `PATH` | 便携 bin、npm-global、node_modules bin 路径 |

这些环境变量只对当前启动进程和它的子进程有效，不会永久写入系统环境变量。

## Provider 配置

默认包不自动写 provider。需要手动添加时，可以编辑：

```text
config\opencode\opencode.json
```

OpenCode 支持 provider 配置，并支持在配置中引用环境变量，例如：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": [
    "oh-my-openagent"
  ],
  "provider": {
    "example": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Example Provider",
      "options": {
        "baseURL": "https://example.com/v1",
        "apiKey": "{env:EXAMPLE_API_KEY}"
      },
      "models": {
        "example-model": {
          "name": "Example Model"
        }
      }
    }
  }
}
```

实际 `baseURL`、模型名和 provider ID 应以对应供应商和 OpenCode 官方文档为准。

## 项目级配置

项目目录可以包含：

```text
项目目录\.opencode\
```

常见项目级配置：

```text
项目目录\.opencode\skills\
项目目录\.opencode\oh-my-openagent.json
项目目录\.opencode\oh-my-openagent.jsonc
```

当使用 `opkcode` 在项目目录启动时，项目级配置更容易被当前任务使用。

## 不建议的配置方式

不建议：

- 直接运行 `bin\opencode.exe`。
- 把 API Key 明文写入 `opencode.json`。
- 删除 `plugin: ["oh-my-openagent"]` 后又期待 OMO 生效。
- 删除 `tui.json` 后又期待 OMO TUI 侧边栏生效。
- 手动创建空的 OMO 配置文件覆盖官方安装器生成的配置。
