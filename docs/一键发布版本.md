# 一键发布版本

本工具用于在 GitHub 仓库中自动递增版本号、提交版本更新、创建 Git tag、生成发行 zip，并把 zip 作为 GitHub Release Asset 上传。

## 入口文件

根目录提供两个入口：

```bat
一键发布版本.cmd
release.cmd
```

`release.cmd` 是纯英文备用入口。两个入口调用同一个 PowerShell 脚本：

```text
scripts\70-release-version.ps1
```

## 发布前要求

需要安装并配置：

```bat
git --version
gh --version
gh auth status
```

如果 GitHub CLI 没有登录，请先运行：

```bat
gh auth login
```

## 版本递增规则

版本号格式为：

```text
major.minor.patch
```

递增规则：

```text
1.0.0 -> 1.0.1
1.1.2 -> 1.1.2
1.1.2 -> 1.1.2
1.9.9 -> 2.0.0
```

只有检测到上一个 tag 之后有新提交，或者当前工作区有未提交改动时，脚本才会继续发布。

## 打包规则

发行包从当前工作区文件直接打包，不使用 `git archive`。

这样可以保持本地文件的真实字节内容，包括：

- PowerShell 脚本的 BOM
- CMD 文件的 ASCII 编码
- CRLF / LF 换行
- Markdown 文件的原始编码

发行包输出到：

```text
.release\
```

`.release\` 会被加入 `.gitignore`，不会提交到仓库。

## 配置文件

配置文件是：

```text
release.conf
```

常用字段：

```ini
PROJECT_NAME=opencode-pocket-kit
VERSION_FILE=VERSION
RELEASE_DIR=.release
RELEASE_NOTES_MODE=git
SUPPRESS_GIT_LINE_ENDING_WARNINGS=1
PACKAGE_SOURCE=working-tree
```

## 发行说明

默认使用 Git 提交记录作为 Release notes。

如果需要手动修改发行说明，可以在 GitHub Release 页面中编辑。


## Release 资产上传策略

本脚本会先生成 `.release` 目录下的发行包，然后使用 `gh release create` 创建 Release，再使用 `gh release upload --clobber` 上传 zip 资产。即使 Release 已存在，也会重新上传并覆盖同名资产。发布完成后会再次读取 Release 资产列表，确认发行包已出现在 GitHub Release 页面中。
