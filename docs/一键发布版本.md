# 一键发布版本脚本

本脚本用于在 GitHub 仓库中自动递增版本号、提交版本文件、创建 tag、生成发行 zip 包，并创建 GitHub Release。

## 使用方式

在仓库根目录运行：

```bat
一键发布版本.cmd
```

也可以运行英文入口：

```bat
release.cmd
```

## 发布规则

脚本会检查上一个 `vX.Y.Z` 标签之后是否存在新的 Git 提交，或者当前工作区是否存在未提交更改。

如果没有新提交，也没有未提交更改，脚本不会发布新版本。

版本号递增规则：

```text
1.0.0 -> 1.0.3
1.0.9 -> 1.1.0
1.9.9 -> 2.0.0
```

## 发行包

发行包会生成在：

```text
.release\
```

该目录会被加入 `.gitignore`，不会进入 Git 提交。

脚本使用 `git archive` 从当前提交生成发行包，因此不会把 `.git`、`.release`、Git 未跟踪文件或本地构建缓存打进 zip。

## GitHub Release

脚本依赖 GitHub CLI：

```bat
gh auth login
```

创建 Release 时会上传发行 zip。如果 Release 已存在，脚本会使用 `gh release upload --clobber` 覆盖对应资产。

## 配置

可编辑根目录 `release.conf`。

常用字段：

| 字段 | 说明 |
|---|---|
| `PACKAGE_NAME` | 发行包文件名前缀 |
| `AUTO_COMMIT_CHANGES` | 是否自动提交未提交更改 |
| `RELEASE_NOTES_MODE` | 发布说明来源，可选 `git`、`manual`、`both` |
| `AUTO_PUSH` | 是否推送分支和 tag |
| `CREATE_GITHUB_RELEASE` | 是否创建 GitHub Release |
| `RELEASE_DIR` | 发行包输出目录 |
| `REMOVE_TRACKED_RELEASE_ZIPS` | 是否从 Git 跟踪中移除误加入仓库的发行 zip |

## 编码说明

根目录 `.cmd` 入口脚本保持 ASCII、CRLF、无 BOM，避免 Windows CMD 把中文路径或脚本名解析出错。

PowerShell 实现脚本使用 UTF-8 with BOM。
