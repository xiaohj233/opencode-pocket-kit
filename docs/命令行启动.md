# 命令行启动

OpenCode Pocket Kit 支持注册一个独立命令，让你像使用 `opencode` 一样在任意终端中启动便携版 OpenCode，但不会占用 `opencode` 这个命令名。

## 默认命令名

默认命令名是：

```bat
opkcode
```

它来自根目录配置文件：

```text
command.conf
```

默认内容：

```ini
COMMAND_NAME=opkcode
```

如果你担心命令名冲突，可以先修改 `COMMAND_NAME`，例如：

```ini
COMMAND_NAME=opk-ai
```

然后再运行注册脚本。

## 注册命令

双击根目录：

```bat
注册命令.cmd
```

它会完成两件事：

1. 在 `cmd/` 目录生成命令启动器，例如 `cmd/opkcode.cmd`。
2. 把 `cmd/` 目录加入当前 Windows 用户的 `Path` 环境变量。

注册后请重新打开一个终端窗口。

## 在当前目录启动 OpenCode

进入任意项目目录：

```bat
cd D:\Code\my-project
opkcode
```

此时 OpenCode 的工作目录就是当前终端所在目录：

```text
D:\Code\my-project
```

这和双击 `启动.cmd` 不同：

| 启动方式 | 工作目录 |
|---|---|
| 双击 `启动.cmd` | 默认 `projects/` 目录 |
| 终端执行 `opkcode` | 当前终端所在目录 |

## 注销命令

如果不想继续在命令行中使用该别名，双击：

```bat
注销命令.cmd
```

它会从当前用户的 `Path` 环境变量中移除本包的 `cmd/` 目录。

## 能否访问工作目录外的文件

OpenCode 进程继承当前 Windows 用户的文件权限。理论上，只要当前用户有权限，绝对路径指向的文件可以被访问。

但代码代理通常以当前工作目录为主要上下文。为了避免路径误判，建议把终端切换到真正的项目根目录后再运行：

```bat
opkcode
```

如果需要操作其他目录，建议在对话中提供完整绝对路径。
