# 诊断排错

## 运行诊断

双击：

```bat
诊断.cmd
```

诊断会检查：

- 便携环境变量。
- OpenCode 配置文件。
- OMO 路由配置候选。
- node、npm、git、opencode、gh。
- OpenCode 版本。
- OpenCode 可见模型。
- OMO Doctor。
- Comment Checker。

## 日志位置

日志位于：

```text
logs```

常见日志：

| 日志 | 说明 |
|---|---|
| `install-*.log` | 安装或更新日志 |
| `network-*.log` | 下载可选组件日志 |
| `doctor-*.log` | 诊断日志 |

## OMO 没有生效

检查：

1. `config\opencode\opencode.json` 是否包含 `plugin: ["oh-my-openagent"]`。
2. `config\opencode	ui.json` 是否包含 `plugin: ["oh-my-openagent/tui"]`。
3. `config\opencode
ode_modules\oh-my-openagent` 是否存在。
4. `npm-global\oh-my-openagent.cmd` 是否存在。
5. 运行 `诊断.cmd` 查看 OMO Doctor 输出。

## 网络下载失败

检查：

1. `proxy.conf` 是否启用了正确的 HTTP 代理端口。
2. 代理软件是否正在运行。
3. GitHub 是否能通过当前代理访问。
4. npm 源是否可访问。
5. 适当提高 `GIT_CLONE_TIMEOUT_SECONDS`。

## CMD 乱码或命令被拆坏

根目录 `.cmd` 文件应保持：

- ASCII。
- CRLF。
- 无 BOM。
- 不写中文提示。

中文输出应放在 PowerShell 脚本中。

## PowerShell 中文乱码

`scripts/*.ps1` 应保持：

- UTF-8 with BOM。

如果手动编辑脚本，请确认编辑器保存格式正确。

## GitHub CLI 未登录

如果诊断显示 GitHub CLI 未登录，可以运行：

```bat
gh auth login
```

这只影响 GitHub 自动化能力，不影响 OpenCode 本体启动。

## 密钥无法解密

可能原因：

- 密码输入错误。
- `vault\secrets.env.enc` 被损坏。
- 文件不是当前格式生成。

解决方式：重新运行 `编辑密钥.cmd` 创建密钥库。
