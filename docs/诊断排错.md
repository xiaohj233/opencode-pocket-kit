# 诊断排错

## 运行诊断

双击：

```bat
诊断.cmd
```

诊断脚本会设置便携环境，然后检查 OpenCode、OMO、TUI、Comment Checker、工具链和配置文件状态。

## 诊断会检查什么

| 检查项 | 说明 |
|---|---|
| 便携环境变量 | `OPENCODE_CONFIG`、`OPENCODE_CONFIG_DIR`、`APPDATA`、`HOME`、`NODE_PATH` 等 |
| 配置文件 | `opencode.json`、`tui.json`、`package.json` |
| OMO 路由配置候选 | 全局和项目级 OMO 配置文件 |
| 工具链 | node、npm、git、opencode、gh、bun、bunx |
| OpenCode 版本 | 调用便携版 `opencode.exe --version` |
| OMO 配置状态 | 检查 OpenCode 插件和 TUI 插件是否写入 |
| Comment Checker | 检查可执行文件和 npm bin 是否存在 |
| OMO Doctor | 调用本地 OMO CLI 的 doctor 命令 |

## 日志位置

日志位于：

```text
logs\
```

常见日志：

| 日志 | 说明 |
|---|---|
| `install-*.log` | 安装或更新日志 |
| `network-*.log` | 下载可选组件日志 |
| `doctor-*.log` | 诊断日志 |

安装脚本可能会清理旧日志，具体行为由 `proxy.conf` 中的 `CLEAN_OLD_LOGS_ON_INSTALL` 控制。

## OMO 没有生效

检查：

1. `config\opencode\opencode.json` 是否包含：

   ```json
   "plugin": [
     "oh-my-openagent"
   ]
   ```

2. `config\opencode\tui.json` 是否包含：

   ```json
   "plugin": [
     "oh-my-openagent/tui"
   ]
   ```

3. 是否存在：

   ```text
   config\opencode\node_modules\oh-my-openagent\
   ```

4. 是否存在：

   ```text
   npm-global\oh-my-openagent.cmd
   ```

5. 运行：

   ```bat
   诊断.cmd
   ```

   查看 OMO Doctor 输出。

如果缺失插件配置，重新运行：

```bat
安装.cmd
```

## 网络下载失败

检查：

1. `proxy.conf` 是否启用了正确的 HTTP 代理端口。
2. 代理软件是否正在运行。
3. GitHub 是否能通过当前代理访问。
4. npm 源是否可访问。
5. 适当提高 `GIT_CLONE_TIMEOUT_SECONDS` 和 `NPM_PACKAGE_TIMEOUT_SECONDS`。
6. 查看 `logs\network-*.log`。

如果网络一直失败，OpenCode 本体可能已经可用，只是部分可选 Skills 没有安装。可以之后网络稳定时再运行：

```bat
更新.cmd
```

## CMD 乱码或命令被拆坏

根目录 `.cmd` 文件应保持：

- ASCII。
- CRLF。
- 无 BOM。
- 不直接写中文提示。

中文输出应放在 PowerShell 脚本中。

如果手动编辑过 `.cmd` 后出现乱码或类似“不是内部或外部命令”的错误，建议从发布包重新覆盖对应 `.cmd` 文件。

## PowerShell 中文乱码

`scripts\*.ps1` 应保持：

- UTF-8 with BOM。

如果手动编辑脚本，请确认编辑器保存格式正确。Windows PowerShell 5.1 对无 BOM 的 UTF-8 脚本支持不稳定。

## GitHub CLI 未登录

如果诊断显示 GitHub CLI 未登录，可以运行：

```bat
gh auth login
```

这只影响 GitHub 自动化能力，不影响 OpenCode 本体启动。

## 密钥无法解密

可能原因：

- 密码输入错误。
- `vault\secrets.env.enc` 被损坏。
- 文件不是当前格式生成。
- 文件被手动编辑或传输损坏。

解决方式：重新运行：

```bat
编辑密钥.cmd
```

创建新的密钥库。

## 启动后找不到 API Key

检查：

1. 是否通过 `启动.cmd` 或 `opkcode` 启动。
2. 是否直接运行了 `bin\opencode.exe`。
3. `vault\secrets.env.enc` 是否存在。
4. 启动时是否输入了正确密码。
5. `编辑密钥.cmd` 中变量名是否写错。

例如 Tavily 应写成：

```env
TAVILY_API_KEY=tvly-xxxx
```

## opkcode 无法使用

检查：

1. 是否运行过：

   ```bat
   注册命令.cmd
   ```

2. 是否重新打开了终端。
3. `command.conf` 中的 `COMMAND_NAME` 是否符合命名规则。
4. 当前用户 `Path` 是否包含本包的 `cmd\` 目录。

## 工作目录不对

双击 `启动.cmd` 默认进入：

```text
projects\
```

命令行运行 `opkcode` 会进入当前终端目录。

如果希望在某个项目里工作，请使用：

```bat
cd D:\Code\my-project
opkcode
```

## 配置损坏

如果 `opencode.json` 被手动写坏，可以先恢复为最小配置：

```json
{
  "$schema": "https://opencode.ai/config.json",
  "plugin": [
    "oh-my-openagent"
  ]
}
```

然后运行：

```bat
安装.cmd
诊断.cmd
```
