# 密钥安全

本项目不要求把 API Key 写入系统环境变量，也不要求把 API Key 明文写入 OpenCode 的配置文件。

## 密钥库位置

加密密钥库位于：

```text
vault\secrets.env.enc
```

示例输入格式见：

```text
vault\secrets.example.env
```

## 创建密钥库

运行：

```bat
编辑密钥.cmd
```

按 `KEY=VALUE` 格式输入密钥：

```env
DEEPSEEK_API_KEY=sk-xxxx
OPENROUTER_API_KEY=sk-or-xxxx
OPENAI_API_KEY=sk-xxxx
```

输入空行结束，然后设置密钥库密码。

## 运行时如何使用密钥

运行：

```bat
启动.cmd
```

如果存在 `vault\secrets.env.enc`，启动脚本会要求输入密码。解密成功后，脚本只会把密钥设置到当前 PowerShell 进程环境变量中，然后启动 OpenCode。

这些变量不会写入系统环境变量，也不会写入用户环境变量。

## 加密格式

当前密钥库格式为：

```text
OCENV002 + salt + iv + ciphertext + hmac
```

加密逻辑：

| 项目 | 说明 |
|---|---|
| 密钥派生 | PBKDF2，200000 次迭代 |
| 派生长度 | 64 字节 |
| 加密密钥 | 前 32 字节 |
| HMAC 密钥 | 后 32 字节 |
| 加密算法 | AES-256-CBC |
| 填充 | PKCS7 |
| 完整性校验 | HMAC-SHA256 |
| salt | 16 字节随机值 |
| iv | 16 字节随机值 |

## 安全边界

这种设计主要防止：

- U 盘丢失后 API Key 被直接读取。
- 配置文件中出现明文 API Key。
- 系统环境变量被永久污染。

它不能防止：

- 当前电脑已经存在恶意软件。
- 运行时进程环境变量被同用户恶意进程读取。
- 密码过弱导致暴力破解。
- 键盘记录器记录输入密码。

## 建议

- 使用较长的密钥库密码。
- 不要把 `vault\secrets.env.enc` 提交到 Git 仓库。
- 发布项目前检查 `.gitignore` 是否生效。
- 在不可信电脑上谨慎输入 API Key 和密钥库密码。
