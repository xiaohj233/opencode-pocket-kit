# 密钥安全

OpenCode Pocket Kit 不要求把 API Key 写入系统环境变量，也不要求把 API Key 明文写入 OpenCode 配置文件。

密钥通过 `编辑密钥.cmd` 加密保存，并在启动时临时注入当前 OpenCode 进程。

## 密钥库位置

加密密钥库位于：

```text
vault\secrets.env.enc
```

示例输入格式见：

```text
vault\secrets.example.env
```

## 创建密钥库

运行：

```bat
编辑密钥.cmd
```

按 `KEY=VALUE` 格式输入密钥，每行一个变量：

```env
DEEPSEEK_API_KEY=sk-xxxx
OPENROUTER_API_KEY=sk-or-xxxx
TAVILY_API_KEY=tvly-xxxx
```

输入空行结束，然后设置密钥库密码。

## 修改密钥库

如果需要修改密钥库，重新运行：

```bat
编辑密钥.cmd
```

然后输入完整的新变量列表。

注意：当前编辑方式是重新生成密钥库，不是增量编辑。也就是说，如果原来有三行变量，重新编辑时仍然需要把三行都输入进去。

## 运行时如何使用密钥

运行：

```bat
启动.cmd
```

或使用命令行入口：

```bat
opkcode
```

如果存在 `vault\secrets.env.enc`，启动脚本会要求输入密码。解密成功后，脚本只会把密钥设置到当前 PowerShell 进程环境变量中，然后启动 OpenCode。

这些变量不会写入系统环境变量，也不会写入用户环境变量。

## 加密格式

当前密钥库格式为：

```text
OCENV002 + salt + iv + ciphertext + hmac
```

加密逻辑：

| 项目 | 说明 |
|---|---|
| 密钥派生 | PBKDF2，200000 次迭代 |
| 派生长度 | 64 字节 |
| 加密密钥 | 前 32 字节 |
| HMAC 密钥 | 后 32 字节 |
| 加密算法 | AES-256-CBC |
| 填充 | PKCS7 |
| 完整性校验 | HMAC-SHA256 |
| salt | 16 字节随机值 |
| iv | 16 字节随机值 |
| 文件标识 | `OCENV002` |

文件结构：

```text
magic(8) + salt(16) + iv(16) + ciphertext(n) + hmac_sha256(32)
```

## 为什么不用明文 .env

明文 `.env` 方便但风险较高：

- U 盘丢失后，API Key 可直接被读取。
- 项目发布或提交 Git 时容易误带密钥。
- 多台电脑之间移动时容易遗留明文文件。

OpenCode Pocket Kit 使用加密密钥库，主要目标是降低“便携介质丢失后密钥直接泄露”的风险。

## 安全边界

这种设计主要防止：

- U 盘丢失后 API Key 被直接读取。
- 配置文件中出现明文 API Key。
- 系统环境变量被永久污染。
- 将项目上传到 GitHub 时误提交明文密钥。

它不能防止：

- 当前电脑已经存在恶意软件。
- 运行时进程环境变量被同用户恶意进程读取。
- 密码过弱导致暴力破解。
- 键盘记录器记录输入密码。
- 你手动把 API Key 粘贴到聊天或日志中。

## 建议

- 使用较长的密钥库密码。
- 不要把 `vault\secrets.env.enc` 提交到 Git 仓库。
- 不要把 API Key 写入 `opencode.json`。
- 不要把明文 `.env` 长期放在 U 盘中。
- 在不可信电脑上谨慎输入 API Key 和密钥库密码。
- 发布项目前检查 `.gitignore` 是否生效。

## 常见变量名

常见模型供应商和工具变量：

```env
DEEPSEEK_API_KEY=sk-xxxx
OPENROUTER_API_KEY=sk-or-v1-xxxx
OPENAI_API_KEY=sk-xxxx
ANTHROPIC_API_KEY=sk-ant-xxxx
TAVILY_API_KEY=tvly-xxxx
GITHUB_TOKEN=ghp_xxxx
```

Tavily 和 GitHub Token 属于工具密钥，不是 OpenCode provider 配置本身。通常只需要作为环境变量注入。
