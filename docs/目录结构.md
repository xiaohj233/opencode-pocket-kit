# 目录结构

本项目把 OpenCode 的便携运行环境集中在一个目录中。文件夹名称保持英文，便于脚本和第三方工具稳定识别。

## 根目录文件

| 路径 | 作用 |
|---|---|
| `安装.cmd` | 安装 OpenCode、OMO、TUI、Comment Checker 和可选 Skills |
| `启动.cmd` | 启动便携版 OpenCode，默认工作目录为 `projects\` |
| `注册命令.cmd` | 注册命令行别名，默认生成 `opkcode` 命令 |
| `注销命令.cmd` | 注销命令行别名目录 |
| `编辑密钥.cmd` | 创建加密密钥库 |
| `更新.cmd` | 更新本体、插件和可选组件 |
| `诊断.cmd` | 检查环境、配置和依赖 |
| `proxy.conf` | 下载代理和安装行为配置 |
| `command.conf` | 命令行别名配置，默认 `COMMAND_NAME=opkcode` |
| `README.md` | 项目主页说明 |
| `VERSION` | 当前发布版本 |
| `LICENSE` | 许可证 |
| `.gitignore` | Git 忽略规则 |
| `.gitattributes` | Git 文本和编码规则 |

## 运行目录

| 目录 | 作用 |
|---|---|
| `bin\` | 便携版 OpenCode 可执行文件所在目录，安装后通常包含 `opencode.exe` |
| `cmd\` | 命令行别名启动器目录，可加入当前用户 `Path` |
| `config\` | 便携配置根目录，启动时会映射为 `APPDATA` |
| `config\opencode\` | OpenCode 配置目录，启动时会映射为 `OPENCODE_CONFIG_DIR` |
| `config\opencode\node_modules\` | OpenCode 配置目录内的 Node 依赖 |
| `config\opencode\skills\` | 全局 Skills 安装目录 |
| `config\opencode\plugins\` | 插件文件目录 |
| `npm-global\` | 便携 npm 全局安装目录 （存储空间紧张时可考虑删除，不影响使用）|
| `projects\` | 双击 `启动.cmd` 时默认进入的项目工作目录 |
| `vault\` | 加密密钥库目录 |
| `logs\` | 安装、更新、下载、诊断日志目录 |
| `repo-cache\` | 可选 Skills 的 Git 仓库缓存 |
| `cache\` | 便携缓存目录 |
| `data\` | 预留数据目录 |
| `home\` | 便携 HOME / USERPROFILE 目录 |
| `backups\` | 配置备份目录 |
| `.portable-state\` | 临时运行状态目录 |
| `scripts\` | PowerShell 实现脚本目录 |
| `docs\` | 项目文档目录 |

## OpenCode 配置目录

核心配置位于：

```text
config\opencode\
```

常见文件：

| 文件或目录 | 作用 |
|---|---|
| `opencode.json` | OpenCode 主配置，默认包含 OMO 插件入口 |
| `tui.json` | OpenCode TUI 配置，默认包含 OMO TUI 插件入口 |
| `package.json` | 配置目录内的 Node 依赖声明 |
| `oh-my-openagent.json` | OMO 官方安装器生成的路由配置 |
| `node_modules\` | OMO、Comment Checker、OpenCode 插件 SDK 等依赖 |
| `skills\` | 全局 Skills 目录 |
| `plugins\` | 插件文件目录 |
| `superpowers\` | Superpowers 仓库内容副本 |

## scripts 目录

| 文件 | 作用 |
|---|---|
| `00-公共函数.ps1` | 公共路径、代理、npm、Git、密钥、配置函数 |
| `10-安装.ps1` | 安装和更新 OpenCode、OMO、依赖 |
| `20-启动.ps1` | 设置便携环境并启动 OpenCode |
| `30-编辑密钥.ps1` | 创建 OCENV002 加密密钥库 |
| `40-诊断.ps1` | 诊断环境和插件状态 |
| `50-下载可选组件.ps1` | 网络测试和可选 Skills 下载 |
| `60-注册命令.ps1` | 生成并注册命令行别名 |
| `61-注销命令.ps1` | 从当前用户 `Path` 移除命令行别名目录 |

## 双击启动与命令行启动的目录差异

| 方式 | 工作目录 |
|---|---|
| `启动.cmd` | `projects\` |
| `opkcode` | 当前终端所在目录 |

如果你只在一个项目中使用，可以把项目放进：

```text
projects\
```

如果你经常在多个项目间切换，建议使用 `opkcode`。

## 不建议手动删除的目录

以下目录由脚本管理，不建议随意删除：

| 目录 | 原因 |
|---|---|
| `config\opencode\` | 存放 OpenCode、OMO、TUI、Skills 和依赖配置 |
| `npm-global\` | 存放便携 npm 全局包 |
| `vault\` | 存放加密密钥库 |
| `bin\` | 存放便携 OpenCode 可执行文件 |
| `cmd\` | 存放命令行别名启动器 |
| `repo-cache\` | 存放下载到本地的可选组件仓库缓存 |

如果需要重新安装，可以先备份 `vault\` 和自己的项目目录，再运行 `安装.cmd` 或 `更新.cmd`。

## 文件编码约定

| 类型 | 编码 | 说明 |
|---|---|---|
| `.cmd` | ASCII、CRLF、无 BOM | 避免 Windows CMD 中文乱码和命令解析问题 |
| `.ps1` | UTF-8 with BOM | 允许 PowerShell 5.1 正确读取中文 |
| `.md` | UTF-8 | GitHub 文档 |
| `.conf` | UTF-8 或 ASCII | 简单键值配置 |
| `.json` / `.jsonc` | UTF-8 | OpenCode、TUI 和 OMO 配置 |
